Gravity

While gravity might not be the most interesting of topics, it is something that will allow us to talk about important physics concecpts, and is something that you experience in your everyday-life. The way that things are positioned on the sceen is very similar to an x-y plane. The screen is mapped out by pixels, so when we want to position something on the screen, we use pixels witin the range 0 to height for the y direction and 0 to width for the x direction. In order to move an object on the screen, we simply need to change the x and y position of the object, remove the old object and display the new one. By doing this very quickly, we can simulate animation. This adjusting of the position of the position of the object is what's known as velocity. While the units of measurement for position are pixels, the units of velocity are pixels per frame, or the number of pixels that the object moves in every unit of time. So, we can vary the position of the object gradually by repeatedly adding the velocity to the position. Where this gets a bit confusing if you have a math or physics background, is since our range is from 0 to height, in order to move the object down on the screen, you need a positive velocity. Now let's take this idea of position and velocity one step further by changing the velocity by a constant amount each frame a.k.a. acceleration. So, for each frame, we add the velocity to the position, and we add the acceleration to the velocity. This is important because gravity is simply an acceleration on an object. Here on Earth, the "acceleration constant" of gravity is 9.8m/s2 (meters per second squared). Unless you have a screen taller than 10 meters, your object would travel the entire height of your screen in one frame so this number isn't very important for us. We are interested in pixels per frame squared (p/f2). You don't have to understand this unit, but understand how it changes the velocity each frame.
